tags

Bismillah preprocessor uses an intern language, guess what? Also called Bismillah.

This tag system allows developers to write C/C++ code whenever they want.

Bismillah preprocessor allows you to use as many Bismillah blocks as you want.

Example
#include <iostream>
using namespace std;

int main() {
    // cpp code here...

    <bismillah>
        # bismillah code here...
    </bismillah>

    // cpp code here...
}
Translation
#include <iostream>
using namespace std;

int main() {
    // cpp code here...

    // a bismillah block started here...
        // bismillah code here...
    // a bismillah code ended here...

    // cpp code here...
}

variables

Based on TypeScript, Bismillah looks so similar when variable initialization is about.

Bismillah defines six basic data types (char, bool, int, float, number & double).

⚠️ If Bismillah does not know of any variable it will be translated anyways. This is because that variable could be initialized outside Bismillah.

⚠️ Bismillah preprocessor does not verify arrays indexes, so you will be notified that an index could be wrong.

Example
<bismillah>
    # variable initialization
    a:int = 1234
    b:int = 4321

    # value assignation
    a = b + a + c[i + j]
    b = a * 2
</bismillah>
Translation
// a bismillah block started here...
    // variable initialization
    int a = 1234;
    int b = 4321;

    // value assignation
    a = b + a + c[i+j];
    b = a * 2;
// a bismillah block ended here...

conditionals

Bismillah tries to give the developer a Python-like experience.

Example
<bismillah>
    a:int = 42
    b:int = a and a or not a
</bismillah>
Translation
// a bismillah block started here...
    int a = 42;
    int b = a && a || ! a;
// a bismillah block ended here...

vectors

As you may know, C++ uses heap and stack memory, Bismillah simplifies this by using a TypeScript syntax based.

The difference between Bismillah vectors is minimum so you don't have to memorize a complex/weird syntax.

Example
<bismillah>
    length:int = 5000

    # stack memory
    heap_vector:number[length]

    # heap memory
    stack_vector:number = [length]
</bismillah>
Translation
// a bismillah block started here...
    int length = 5000;

    // stack memory
    float heap_vector[length];

    // heap memory
    float *stack_vector = (float *) malloc(length * sizeof(float));
// a bismillah block ended here...

cuda vectors

You are getting into dark places, CUDA vectors seem to be quite difficult, but that's why Bismillah exists... to make your life a little bit easier.

Forget about pointers 😏.

Example
<bismillah>
    length:int = 314159

    # reserved on device
    cuda dev_a:number = [length]
</bismillah>
Translation
// a bismillah block started here...
    int length = 314159;

    // reserved on device
    float *dev_a;
    cudaMalloc((void **)&dev_a, length * sizeof(float));
// a bismillah block ended here...

free

If you reserve memory, you have to free it after using it. But don't worry, you don't have to do it in the C++ way (freeing vectors one by one) cause doing it that way is such an old fashioned thing, just like love 💔.

🔴 Bismillah must know vectors, so if you wanna use the free_all function then you should create your vectors inside a Bismillah block.

Example
<bismillah>
    length:int = 5000

    # heap memory
    a:number = [length]
    b:number = [length]
    c:number = [length]

    # freeing everything <3
    free_all()
</bismillah>
Translation
// a bismillah block started here...
    int length = 5000;

    // heap memory
    float *a = (float *) malloc(length * sizeof(float));
    float *b = (float *) malloc(length * sizeof(float));
    float *c = (float *) malloc(length * sizeof(float));

    // freeing everything <3
    free(a);
    free(b);
    free(c);
// a bismillah block ended here...

cuda free

If you reserve memory on GPU, you must free it.

🔴 Just like the free_all function, if you wanna use the cuda_free_all function you need to create your CUDA vectors inside a Bismillah block.

Example
<bismillah>
    length:int = 5000

    # heap memory
    cuda a:number = [length]
    cuda b:number = [length]
    cuda c:number = [length]

    # freeing everything <3
    free_all()
</bismillah>
Translation
// a bismillah block started here...
    int length = 5000;

    // heap memory
    float *dev_a;
    cudaMalloc((void **)&dev_a, length * sizeof(float));
    float *dev_b;
    cudaMalloc((void **)&dev_b, length * sizeof(float));
    float *dev_c;
    cudaMalloc((void **)&dev_c, length * sizeof(float));

    // freeing everything <3
    cudaFree(dev_a);
    cudaFree(dev_b);
    cudaFree(dev_c);
// a bismillah block ended here...
